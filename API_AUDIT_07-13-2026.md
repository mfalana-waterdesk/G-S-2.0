## Architecture — What's Built

### Endpoint
```
POST https://purewaterpartnerstest2.leaseteam.net/LeaseTeam.Aspire.Api/1/contract/scheduledbillables
```

### Call URL Action
- **Name:** Send Scheduled Billable to Aspire
- **Table:** Dealer Goods & Services (fires from parent level)
- **Trigger:** Custom button on G&S form ("SEND SCHEDULED BILLABLE TEST")
- **Pattern:** One API call per G&S record, combining all child fees

## Aspire API Reference (Validated)

### Endpoint
```
POST https://purewaterpartnerstest2.leaseteam.net/LeaseTeam.Aspire.Api/1/contract/scheduledbillables
```

### Field Formats (confirmed working)
| Field | Format |
|-------|--------|
| TransactionCode | `{"Code": "CLEANING"}` |
| ContractId | `{"Value": "104762", "Type": "transaction"}` — must be numeric |
| AssetRecordId | `{"Value": "APIE-93067", "Type": "record"}` |
| ProrationMethod | `{"Code": 1, "Description": "EquipmentCost"}` |
| Dates | `"yyyy-MM-dd"` format (e.g., "2026-06-29") |
| Amounts | Numeric (no quotes): `100`, `25.50` |
| Occurrences | Numeric: `1`, `3` |
| FollowRent | Always `false` (dropped from scope) |
| SuspendInvoicing | Always `true` (during testing/rollout) |

### Validation Rules (learned from errors)
- ContractId.Value MUST be numeric when Type="transaction"
- Asset FirstPaymentDate MUST be within the Payment date range (≥ StartDate)
- TransactionCode.Code must be a valid code in Aspire Admin (CLEANING, PM, LIFT, TRIP, etc.)
- Asset PaymentAmounts must sum to Payments.Amount
